VerBump

Versions-Manager für Windows-Entwickler

Alle VERSION-Dateien deiner Projekte im Blick. Major, Minor oder Patch mit einem Tastendruck erhöhen — SemVer, CalVer oder ein eigenes Schema.

⬇ Aktuelle Version herunterladen Auf GitHub ansehen

Windows 10/11 · Eigenständige EXE · Kein Installer · Kostenlos & Open Source

Warum VerBump?

Ich arbeite an etlichen Projekten gleichzeitig und habe schon die unterschiedlichsten Dinge in Zusammenhang mit Versionierung ausprobiert. Das Problem ist nicht die Komplexität — ich weiß genau, welche Datei ich editieren muss. Das Problem ist, dass mein Kopf schon drei Schritte weiter ist, wenn ich git push eintippe. VerBump funktioniert, weil es einen Moment des Innehaltens erzwingt: ein Blick, alle Projekte, bevor irgendetwas rausgeht.

Und weil ich eigentlich ein fauler Sack bin, wollte ich das Thema endgültig lösen. Vor dem git push rufe ich einfach VerBump auf und habe auf einen Blick den Status aller Repos im Blick — Orange bedeutet: hier wurde etwas geändert, aber die Version noch nicht erhöht.

Das spart mir mehr Sorgen, Nacharbeit und Zeit, als ich für die Entwicklung dieser Lösung investiert habe.

— Michael Baas, Entwickler von VerBump

Features

⌨

Tastaturgesteuert

Mit Alt+A–Z oder Alt+0–9 direkt zum Projekt springen. Version mit Strg+1–4 erhöhen. Ohne Maus.

📋

Mehrere Schemas

Unterstützt SemVer, CalVer und vollständig anpassbare sequentielle Schemas — pro Projekt konfigurierbar.

🔍

Aktualitätsprüfung

Scannt Projektverzeichnisse automatisch und markiert Projekte, bei denen Quelldateien neuer als die aktuelle VERSION sind.

⚙

Flexible Ignore-Regeln

Globale Ignore-Listen für Verzeichnisse und Dateimuster, mit projektspezifischen Überschreibungen per !-Präfix-Syntax.

🌐

Mehrsprachig

Lieferbar mit Deutsch und Englisch. Weitere Sprachen lassen sich durch eine lang.xx.json-Datei neben der EXE ergänzen.

📦

Keine Abhängigkeiten

Eine einzige, eigenständige .exe — keine .NET-Installation, keine Registry-Einträge, kein Installer erforderlich.

Dokumentation

Erste Schritte

  1. VerBump.exe herunterladen und beliebig ablegen.
  2. Starten — beim ersten Start wird settings.json in %APPDATA%\VerBump\ angelegt.
  3. Einstellungen öffnen (Schaltfläche Einstellungen) und Projekte hinzufügen.
  4. Jedes Projekt benötigt eine einfache Text-Datei VERSION im Stammverzeichnis.

Versionsschemas

SchemaBeispielBump-SchaltflächenHinweis
semver1.4.2 Major · Minor · Patch Standard. Niedrigere Teile werden auf 0 zurückgesetzt, wenn „Reset on bump" aktiviert ist.
calver25.03.1 Datumsteile automatisch · numerische Teile manuell Format konfigurierbar, z. B. YY.MM.patch oder YYYY.MM.DD.build.
sequentialmajor.minor.patch Eine Schaltfläche pro Teil Vollständig anpassbare Teilnamen über das Format-Feld, z. B. release.build.

Ignore-Regeln

VerBump scannt Projektverzeichnisse auf Dateien, die neuer als die aktuelle VERSION sind. Verzeichnisse und Dateimuster können global oder pro Projekt ausgeschlossen werden.

Global ignore — in den Einstellungen, gilt für alle Projekte:

Dirs:  .git  node_modules  bin  obj  dist
Files: *.bak  *.tmp  *.log

Projektspezifisch — im Bereich „Entry" der Einstellungen. Mit !-Präfix kann ein global ausgeschlossener Eintrag wieder eingeschlossen werden: 

Dirs:  build        ← zusätzlich für dieses Projekt ausschließen
       !bin         ← 'bin' wieder einschließen (überschreibt globale Regel)

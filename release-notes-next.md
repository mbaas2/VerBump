# What's new in vNEXT

## New features

- **Open single VERSION file** — right-click any `VERSION` file → *Open in VerBump*:
  shows only that one project, no central setup needed.
  If the file is not yet in `settings.json`, a **+** toolbar button lets you add it permanently.

## Improvements

- Explorer context menu for `VERSION` files: switched from CommandStore to nested shell keys
  (fixes submenu not opening on some Windows 11 configurations)
- Website: workflow comparison table explaining central vs. single-file vs. multi-settings approaches

## Was ist neu in vNEXT

## Neue Features

- **Einzelne VERSION-Datei öffnen** — Rechtsklick auf eine `VERSION`-Datei → *In VerBump öffnen*:
  zeigt nur dieses eine Projekt, kein zentrales Setup nötig.
  Wenn die Datei noch nicht in `settings.json` steht, ermöglicht ein **+**-Button in der Toolbar
  das dauerhafte Hinzufügen.

## Verbesserungen

- Explorer-Kontextmenü für `VERSION`-Dateien: von CommandStore auf verschachtelte Shell-Keys umgestellt
  (behebt fehlendes Untermenü unter manchen Windows-11-Konfigurationen)
- Website: Workflow-Vergleichstabelle (zentral / einzelne Datei / mehrere Settings-Profile)
